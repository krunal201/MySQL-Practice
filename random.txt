6.2
Unit Testing
Unit testing involves testing individual components or units of code to ensure they function correctly in isolation. Each unit, such as the hotel search functionality, filtering system, or booking form, is tested for expected output based on given input. This is essential for catching errors early in the development cycle.

System Testing
This type of testing involves testing the entire system as a whole. It checks the system’s compliance with the functional and non-functional requirements outlined in the project specifications. System testing simulates how real users would interact with the platform and ensures that all components function correctly when integrated.

Black Box Testing
Black box testing is a software testing method that focuses entirely on testing the functionality of the system, without knowledge of the internal code or architecture. The tester evaluates the platform based on its inputs and outputs, ensuring the system behaves as expected from the end user’s perspective.
